# 01. Быстрый старт: выбор VPS и первое подключение

Выбор VPS — это реально **половина успеха**. Хорошая конфигурация на «грязном» IP
проживёт недолго.

## На что смотреть при покупке VPS

1. **Локация.** Близкие к РФ и быстрые: Финляндия, Германия, Нидерланды, Швеция.
   Чем ближе — тем ниже пинг.
2. **«Чистота» IP.** IP не должен уже быть в чёрных списках РКН. Иногда блокируют
   целые подсети/ASN. Идеально — взять сервер, проверить IP, и если он уже не
   пингуется из РФ — сразу менять.
3. **Возможность сменить IP.** Хорошо, если хостинг позволяет быстро
   переназначить/докупить IP — пригодится, когда (а не «если») IP заблокируют.
4. **Не брать заведомо «спалённые» диапазоны.** Крупные дешёвые облака, чьи
   диапазоны массово используются под VPN, блокируют целиком.

**Характеристики:** 1 vCPU, 512 МБ–1 ГБ RAM, 10 ГБ диска хватает с запасом.
Xray почти не ест ресурсы. **ОС:** чистый Ubuntu 22.04/24.04 или Debian 12 — не
берите готовые «панельные» образы, всё ставим сами. **Стоимость:** ~2–6 €/мес.
Слишком дешёвые «перепроданные» серверы часто имеют «грязные» IP.

**Оплата:** часто нужна зарубежная карта — российские карты у заграничных
хостингов часто не проходят.

## Проверка IP перед боем

После покупки узнайте IP и проверьте доступность из России:

```bash
./scripts/check-ip-russia.sh <IP>
```

Или попросите знакомого из РФ пингануть IP / открыть `https://<IP>`. Если IP уже
не отвечает из РФ — сразу меняйте, не тратьте время на настройку.

## Первое подключение по SSH

```bash
ssh root@ВАШ_IP
# первый раз: yes для подтверждения отпечатка, затем пароль (вводится «вслепую»)

# если хостинг дал ключ:
ssh -i путь_к_ключу root@ВАШ_IP
```

Вы внутри, когда видите приглашение `root@hostname:~#`.

## Развёртывание

```bash
git clone <repo-url> vpn-dimosi && cd vpn-dimosi
cp .env.example .env
nano .env                 # проверьте REALITY_SNI, при необходимости порты/Telegram
sudo ./deploy.sh all
```
