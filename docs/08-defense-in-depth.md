# 08. Оборона глубиной: каталог всех схем

Один протокол на одном сервере — это уже не «навсегда». Нужна **оборона глубиной**:
несколько схем, несколько серверов, готовые резервы. TCP-маскировка
(Reality/XHTTP/CDN) и UDP-протоколы (Hysteria2/AmneziaWG) ломаются **разными
способами** — держите по одному представителю каждого типа.

## Сравнительная карта

| Схема | Транспорт | Сильна против | Когда брать |
|-------|-----------|---------------|-------------|
| **VLESS + Reality + Vision** | TCP/TLS | маскировка под HTTPS | базовый дефолт (этот репо) |
| **VLESS + XHTTP** | TCP/HTTP (можно через CDN) | JA3/JA4 и поведенческий анализ | когда Reality начал лагать |
| VLESS + gRPC | TCP/HTTP2 | похож на обычный gRPC | альтернатива XHTTP |
| **VLESS за CDN Cloudflare** | TCP/TLS через чужой IP | блокировка IP сервера | когда блокируют именно ваш IP |
| **CFXHTTP** (CF Pages/Workers) | на инфраструктуре CF | не нужен ни VPS, ни домен | бесплатный запасной аэродром |
| **Hysteria2** | UDP/QUIC + Salamander | скорость, port hopping | игры/видео; где UDP не душат (этот репо) |
| **AmneziaWG** | UDP (обфусц. WireGuard) | простота, часть белых списков | простой WG-подобный, UDP жив |
| Trojan | TCP/TLS | — | резерв, не основной |
| Shadowsocks-2022 (+ShadowTLS) | TCP | с ShadowTLS лучше | доп. вариант |
| TUIC | UDP/QUIC | скорость | эксперименты |
| **Tor + мосты** | разное | тотальная блокировка всего | последний рубеж |
| Клиентский обход DPI (zapret/GoodbyeDPI) | без туннеля | разблокировка отдельных сайтов | нужен сайт, не туннель |

Подробные пошаговые рецепты для каждой — в исходных гайдах (Часть 3). Ниже —
краткая суть и куда смотреть.

## Схема A. VLESS + XHTTP (следующий шаг после Reality)

XHTTP — новый транспорт Xray, закрывает дыры, через которые ТСПУ ловит классический
VLESS (JA3/JA4, поведенческие тайминги). Если Reality начал «долго коннектиться /
отваливаться» — первый кандидат на переход. Ставится в той же 3X-UI: Inbound с
`network=xhttp`, `security=reality`, **flow оставить пустым** (Vision для xhttp не
используется), `path=/xh`. Нужны свежий Xray-core и свежие клиенты (старые XHTTP не
понимают).

## Схема B. VLESS за CDN Cloudflare

Наружу виден IP Cloudflare, а не ваш сервер → блокировка вашего IP не страшна.
Нужен домен + бесплатный аккаунт Cloudflare. DNS-запись A `vpn.домен → IP VPS`,
оранжевое облако (Proxied), SSL/TLS = Full. Inbound: `vless` + `ws` (WebSocket) +
`tls`, `path=/ws`. CF на free-тарифе проксирует 443/8443/2053/2083. Фишка «чистый
IP»: если IP CF тротлят — в клиенте указать рабочий CF-IP (Address), а SNI/Host =
ваш домен (рабочие IP ищут утилитами cfst/cfray).

## Схема C. CFXHTTP — бесплатный прокси на Cloudflare

Аварийный канал **без VPS и без домена**, бесплатно. Нужен только аккаунт
Cloudflare. Fork проекта CFXHTTP → Cloudflare Pages → задать env-переменные
`UUID` и `XHTTP_PATH`/`WS_PATH` → передеплой. CF выдаст `ваш-проект.pages.dev`,
проект генерирует готовую `vless://` ссылку. Ограничения: скорость/лимиты free CF,
не для тяжёлого трафика. **Сохраните ссылку в оффлайн-чемоданчик.**

## Схема E. AmneziaWG (+ WARP)

Обфусцированный WireGuard — простой, лёгкий, ломает WG-отпечаток, проходит часть
белых списков. Проще всего — десктоп-приложение **AmneziaVPN**: вводите SSH-данные
VPS, оно само ставит сервер, выбираете протокол AmneziaWG, профиль уходит на
телефон по QR. Под мобильные сети: `ListenPort=443`, при лагах `MTU=1280`,
`DNS=1.1.1.1`.

## Self-steal Reality (свой домен)

Маскировка под **свой** реальный сайт → полный контроль над «донором», полезно
для белых списков. nginx отдаёт мини-сайт локально (`127.0.0.1:8443`) с реальным
сертификатом вашего домена; Xray-Reality слушает публичный 443, в `dest` указывает
на этот локальный сайт. Главное правило: **на 443 TLS держит Xray, а не nginx**;
nginx — только локальный донор для проксирования чужаков.

## Последний рубеж: Tor + мосты

Если заблокировано вообще всё (серверы и CDN) — Tor с мостами, которых нет в
публичных списках:
- **snowflake** — временные WebRTC-прокси добровольцев (когда статические мосты
  заблокированы);
- **obfs4** — классический обфусцированный мост;
- **webtunnel** — выглядит как обычный HTTPS-трафик (идейно близко к Reality).

Tor Browser → Settings → Connection → встроенные мосты, или запросить на
bridges.torproject.org. Медленно, но это «связь, когда связи нет». **Держите Tor
Browser установленным заранее** как аварийный комплект.

## Клиентские обходчики DPI (без туннеля)

Когда нужен **конкретный сайт** (YouTube/Discord), а не анонимность. «Дурят» DPI
на вашем устройстве, разбивая/искажая первые пакеты. Свой IP не меняется:
- **GoodbyeDPI** (Windows) — фрагментация пакетов;
- **zapret** (Linux/роутеры/Windows) — мощный, гибкий, активно поддерживается;
- **ByeDPI / Ciadpi** (Android) — локальный прокси с DPI-обходом.

Против белых списков обычно бессильны. Это дополнение к туннелю, не замена.

## Управление многим: Marzban + подписки

Мощная панель для многих протоколов и пользователей с **подписками** (одна ссылка
на пользователя, серверы меняются на бэкенде). На дополнительных VPS ставится
**Marzban-node** → одна панель управляет несколькими странами/IP. Спрячьте
дашборд за nginx + домен + TLS. **3X-UI vs Marzban:** один сервер и простота →
3X-UI; много серверов/пользователей и централизованное управление → Marzban.

## Рекомендованный минимальный боевой комплект

1. **Сервер №1:** Reality (443/tcp) + Hysteria2 (443/udp + hopping) — `sudo ./deploy.sh all`.
2. **Сервер №2** (другой хостинг/страна): то же самое, для отказоустойчивости.
3. **CFXHTTP** как бесплатный резерв.
4. На всех устройствах — **Hiddify** с подпиской/всеми профилями и автопереключением.
5. В «чемоданчике»: zapret/GoodbyeDPI/ByeDPI + Tor Browser + все ссылки/ключи/бэкапы оффлайн.
