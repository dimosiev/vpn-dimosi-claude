# 09. Отладка: логи, типичные ошибки, чеклист

Большинство проблем решаются чтением логов и проверкой по чеклисту.

## Где смотреть

```bash
sudo ./deploy.sh health        # быстрый автоматический healthcheck

# Xray (чистый)
systemctl status xray
journalctl -u xray -f          # логи в реальном времени (Ctrl+C для выхода)
tail -f /var/log/xray/error.log

# Hysteria2
systemctl status hysteria-server
journalctl -u hysteria-server -f

# Панель 3X-UI
x-ui status
journalctl -u x-ui -f

# Слушается ли порт нужным процессом
ss -tlnp | grep 443            # должна быть строка с xray
ss -ulnp | grep 443            # Hysteria2 (UDP)

# Файрвол
ufw status verbose
```

## Универсальный алгоритм диагностики (6 шагов)

1. **Сервер жив?** SSH заходит, `deploy.sh health` показывает active.
2. **Xray слушает 443?** `ss -tlnp | grep 443` показывает `xray`.
3. **Файрвол пускает?** `ufw status` — 443 и порт панели разрешены.
4. **Параметры совпадают?** Сверьте на клиенте и сервере 1:1: UUID, Public Key,
   SNI, Short ID, Flow, Port (`sudo ./scripts/gen-client-link.sh` печатает их).
5. **IP доступен из РФ?** `./scripts/check-ip-russia.sh`. Если нет — это
   блокировка, см. [07-when-blocked.md](07-when-blocked.md).
6. **Логи Xray при попытке подключения** — есть ли ошибки рукопожатия?

Если прошли все 6 и нашли расхождение — почти всегда проблема именно там.

## Типичные проблемы

| Симптом | Вероятная причина | Что делать |
|---------|-------------------|-----------|
| Панель не открывается | порт закрыт в файрволе / неверный путь | `ufw allow ПОРТ`, проверьте URL и web base path, `x-ui status` |
| Клиент «подключено», но нет интернета | не совпали параметры (UUID/pbk/SNI/sid/flow) ИЛИ IP заблокирован | сверьте конфиг 1:1; проверьте доступность IP из РФ |
| Сразу ошибка рукопожатия | неверный SNI/dest или донор без TLS 1.3 | смените донора на `www.microsoft.com`, перезапустите Xray |
| Работает на Wi-Fi, не на мобильном | белые списки оператора | [07: multi-hop](07-when-blocked.md) |
| Скорость ~0 при «подключено» | DPI-замедление / плохой IP | смените IP, проверьте донора, обновите Xray |
| Через пару дней всё «умерло» | заблокирован IP | смена IP, импорт бэкапа |
| Xray не стартует после правки | ошибка в конфиге | `xray run -test -config …`, `journalctl -u xray`, восстановите бэкап |
| Случайные сканеры/боты в логах | это норма | Reality безопасно проксирует их на донора; не паникуйте |

## Частые причины (90%)

Файрвол · несовпадение параметров клиента и сервера · заблокированный IP.

## Обновление (важно для обхода)

DPI эволюционирует — держите Xray-core свежим:

```bash
# чистый Xray (повторная установка обновляет core)
sudo ./scripts/02-install-xray.sh      # переиспользует ключи

# панель: Panel Settings → Xray Version → последняя стабильная
# Hysteria2:
bash <(curl -fsSL https://get.hy2.sh/) && systemctl restart hysteria-server
```

После любого обновления — `sudo ./scripts/backup.sh`.
