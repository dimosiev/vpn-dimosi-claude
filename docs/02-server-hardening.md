# 02. Базовая безопасность сервера

Сервер с публичным IP начинают «долбить» боты в первые минуты. Минимальная
гигиена обязательна. Всё это автоматизирует `scripts/01-harden.sh`.

## Что делает `01-harden.sh`

1. **Обновление системы:** `apt update && apt upgrade -y`.
2. **Отдельный sudo-пользователь** (`ADMIN_USER`, по умолчанию `myadmin`) — чтобы
   не работать под root постоянно. Создаётся без пароля; задайте его сами:
   ```bash
   passwd myadmin
   ```
3. **Файрвол UFW.** Открываются ТОЛЬКО нужные порты:
   - `SSH_PORT/tcp` (22) — **первым**, иначе потеряете доступ при `enable`
   - `VLESS_PORT/tcp` (443) — Reality
   - `PANEL_PORT/tcp` (2053) — панель (если используете)
   - `HY2_PORT/udp` + `HY2_HOP_RANGE/udp` — Hysteria2
4. **fail2ban** — банит IP, перебирающие пароль по SSH.

> ⚠️ Главное правило файрвола: **сначала `ufw allow` нужные порты, потом
> `ufw enable`** — иначе потеряете SSH. Скрипт делает это в правильном порядке.

## Вход по SSH-ключу (рекомендуется)

Сильно повышает безопасность. На своём компьютере:

```bash
ssh-keygen -t ed25519              # если ключа ещё нет
ssh-copy-id myadmin@ВАШ_IP
```

Затем на сервере отключите вход по паролю:

```bash
sudo sed -i 's/^#\?PasswordAuthentication.*/PasswordAuthentication no/' /etc/ssh/sshd_config
sudo systemctl restart ssh
```

> Проверьте, что ключ работает, **до** отключения паролей — иначе закроете себе доступ.

## Защита панели 3X-UI (если используете)

Панель = ключи от сервера. Её нельзя оставлять нараспашку:

1. Смените логин/пароль на длинные уникальные, включите 2FA.
2. Задайте **секретный web base path** (например `/k7zP9xQ2w/`) — панель отвечает
   только по нему, сканеры её не найдут.
3. Нестандартный порт панели (у вас уже так).
4. **Лучший вариант — не выставлять панель в интернет вообще**, а ходить через
   SSH-туннель:
   ```bash
   ssh -L 2053:localhost:ПОРТ_ПАНЕЛИ myadmin@ВАШ_IP
   # панель: http://localhost:2053/ВАШ_ПУТЬ ; порт наружу закрыть:
   sudo ufw delete allow 2053/tcp
   ```

## Важно про nginx/Caddy на 443

Не вешайте обратный прокси, расшифровывающий TLS, **перед** Reality на порт 443.
Reality требует, чтобы именно Xray держал TLS-соединение и сам маскировался под
донора. Если nginx расшифрует трафик до Xray — маскировка рассыпется. Либо
Reality на 443 без прокси, либо разносите по разным IP/портам (см.
[08-defense-in-depth.md](08-defense-in-depth.md), self-steal Reality).
