# 06. Клиенты: Shadowrocket и другие приложения

Нужен **современный прокси-клиент** с поддержкой VLESS + Reality (flow Vision).
Обычные «классические» VPN-приложения тут не подойдут.

| Платформа | Приложение |
|-----------|-----------|
| **iPhone/iPad** | **Shadowrocket** (платное, основное), v2RayTun, Hiddify, Streisand |
| Android | v2RayTun, Hiddify, v2rayNG |
| Windows | v2rayN, Hiddify |
| macOS | Shadowrocket, v2RayTun, Hiddify |

---

## Shadowrocket (iPhone) — пошагово

### Шаг 1. Показать QR на сервере

Зайди на сервер (`ssh root@ТВОЙ_IP`) и выполни:
```bash
sudo ./scripts/gen-client-link.sh        # основной канал Reality (QR + ссылка)
```
Для резервного канала Hysteria2 ссылку `hysteria2://…` печатает `bootstrap.sh` в
самом конце (или загляни в файл `/etc/vpn-dimosi/link-hysteria2.txt`).

### Шаг 2. Добавить профиль в Shadowrocket

**Способ А — по QR (самый надёжный):**
1. Открой Shadowrocket → нажми **«+»** вверху справа.
2. Нажми значок **QR-кода** (справа в поле адреса).
3. Наведи камеру на QR в окне терминала → профиль добавится сам.

**Способ Б — по ссылке:**
1. Скопируй ссылку `vless://…` **прямо из окна терминала** (не из мессенджера!).
2. Открой Shadowrocket — он часто сам предложит **«Добавить из буфера»**. Если нет:
   нажми **«+»** → вставь ссылку в верхнее поле → **Done**.

### Шаг 3. Подключиться

Переключи **тумблер вверху** Shadowrocket в положение **ON** и выбери нужный профиль
(галочка слева от него).

### Шаг 4. Два профиля = основной + резерв

Добавь **оба** канала как отдельные профили:
- `dimosi-reality` — **основной** (VLESS + Reality);
- `dimosi-hy2` — **резерв** (Hysteria2, другой тип канала).

Если основной начал лагать или его «прижали» — переключаешься на резерв **одним
тапом**, не меняя ничего на сервере.

> 💡 **Автовыбор по задержке.** В Shadowrocket можно объединить несколько профилей и
> включить автоматический выбор быстрейшего живого: вкладка **«…» (More)** или при
> создании группы выбери режим **URL-Test / Auto**. Тогда при падении одного канала
> приложение само переключится на другой. Это удобно, когда у тебя несколько серверов.

## Проверка после подключения

1. **IP сменился** — открой `https://ipinfo.io`, должен показать IP сервера, а не твой город.
2. **Нет утечек** — любой «DNS leak test»: без утечек DNS / WebRTC / IPv6.
3. **Скорость адекватная** — прогони спидтест.
4. **Маскировка работает** — открой `https://ТВОЙ_IP` **без** VPN: увидишь страницу
   донора или TLS-ошибку, но не что-то выдающее прокси.

Все 4 пункта ОК → у тебя рабочий устойчивый VPN.

## Несколько серверов и автопереключение (на будущее)

Когда поднимешь второй сервер (в другой стране) — добавь его профили в то же
приложение. Тогда блокировка одной «двери» = переход на другую:
- в **Shadowrocket** — через группу с режимом URL-Test (см. выше);
- в **Hiddify / sing-box** — готовый конфиг с автовыбором генерирует
  `sudo ./scripts/gen-singbox-client.sh` (файл `out/client-singbox.json`).

## Подписки (subscription) — удобство при смене серверов

Вместо рассылки новых ссылок при каждой смене сервера можно дать клиенту **одну
ссылку-подписку**: меняешь серверы на бэкенде — приложение само подтягивает свежие
конфиги. Подписки умеют **3X-UI** и **Marzban** (см.
[08-defense-in-depth.md](08-defense-in-depth.md)); для многих серверов/людей удобнее Marzban.
