# 03. VLESS + Reality + Vision — основной туннель

## Как это работает «под капотом»

1. Клиент инициирует TLS-рукопожатие, как будто открывает сайт-донор (`SNI = www.microsoft.com`).
2. Ваш Xray-сервер подменяет рукопожатие так, что наружу (для DPI) всё выглядит
   как настоящее соединение с microsoft.com — включая реальный TLS-сертификат сайта.
3. Если постучится «случайный» (бот РКН, сканер) без правильных ключей — сервер
   проксирует его на настоящий microsoft.com. Сканер видит настоящий сайт и
   уходит ни с чем. Это защита от **active probing**.
4. «Свой» клиент с правильными ключами получает внутри маскировки реальный туннель.

## Ключевые секреты (должны совпадать на сервере и клиенте)

| Параметр | Что это | Где в нашей инфраструктуре |
|----------|---------|---------------------------|
| **UUID** | идентификатор пользователя | `STATE_DIR/uuid` |
| **Public Key** (Reality) | публичный ключ; клиент его знает | `STATE_DIR/reality_public` |
| Private Key | приватный, остаётся на сервере | `STATE_DIR/reality_private` |
| **Short ID** | отсекает случайных ботов | `STATE_DIR/short_id` |
| **SNI / dest** | имя сайта-донора | `.env`: `REALITY_SNI` / `REALITY_DEST` |
| **Flow** = `xtls-rprx-vision` | защита от анализа размеров пакетов | в шаблоне конфига |
| **Fingerprint** = `chrome` | имитация TLS-отпечатка браузера | задаётся на клиенте |

Если хоть один параметр не совпал — маскировка не срабатывает, сервер ведёт себя
как донор. Это by design.

## Выбор сайта-донора (~80% качества маскировки)

Требования к донору:
- иностранный сервер (вне РФ);
- **не заблокирован** РКН (иначе маскируетесь под уже забаненное);
- поддерживает TLS 1.3 и HTTP/2;
- имеет собственную главную страницу без редиректов;
- крупный и «дорогой для блокировки».

**Хорошие кандидаты:** `www.microsoft.com`, `www.apple.com`, `www.cloudflare.com`,
крупные CDN. **Избегайте** Google-доменов (специфичные cipher suites + периодически
блокируются). Идеально — донор, физически близкий к вашему серверу. Для поиска
доноров «рядом» с VPS есть `RealiTLScanner` (запускать только с одноразового VPS!).

Для новичка достаточно `www.microsoft.com` — надёжный дефолт (он же в `.env.example`).

## Развёртывание

```bash
sudo ./scripts/02-install-xray.sh
```

Скрипт: ставит свежий Xray-core → генерирует UUID, ключи Reality (`xray x25519`),
Short ID → рендерит [`configs/xray/config.template.json`](../configs/xray/config.template.json)
в `/usr/local/etc/xray/config.json` → валидирует (`xray run -test`) → запускает →
печатает vless:// ссылку и QR.

Повторный запуск **переиспользует** сохранённые ключи — старые ссылки не ломаются.

## Параметры максимальной устойчивости (из гайда)

- VLESS + Reality + Vision (`xtls-rprx-vision`), `uTLS = chrome`, порт **443**.
- Качественный «чистый» IP.
- Хороший донор, желательно близкий к серверу.
- Спрятанная панель / чистый Xray.
- Свежий Xray-core (обновляйте — новые версии лучше обходят эвристики).
- Готовность сменить IP/порт/SNI за минуты (бэкап + процедура).
- Маршрутизация: `.ru` и приватные сети — мимо туннеля (уже в конфиге и в
  sing-box-профиле) — снижает нагрузку и делает поведение «естественнее».

## Несколько пользователей / устройств

В чистом Xray добавьте ещё клиентов в массив `clients` конфига (каждому свой
UUID, `flow: xtls-rprx-vision`) — так видно трафик каждого, не открывая новых
портов. Для GUI-управления лимитами/сроками удобнее панель 3X-UI или Marzban
(см. [08-defense-in-depth.md](08-defense-in-depth.md)).
