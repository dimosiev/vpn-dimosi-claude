# 10. Мониторинг с уведомлениями в Telegram

Лучший способ «узнать о блокировке первым» — не ждать, пока «интернет вдруг
пропал», а чтобы **сервер сам проверял себя** каждые 5 минут и **писал тебе в
Telegram**, если что-то не так (IP перестал отвечать из РФ, сервис упал).

Настраивается один раз, дальше работает само. Все команды — **на сервере**
(`ssh root@ТВОЙ_IP`, приглашение с `#`).

---

## Шаг 1. Создать Telegram-бота (2 минуты, бесплатно)

1. В Telegram найди **@BotFather** → отправь команду `/newbot`.
2. Придумай имя и username бота (username должен заканчиваться на `bot`).
3. BotFather пришлёт **токен** — длинную строку вида `123456789:AAH...xyz`. Сохрани её.
4. Найди в Telegram **@userinfobot** → напиши ему что угодно → он пришлёт твой
   **Id** (число, например `987654321`).
5. **Важно:** открой своего нового бота и нажми **Start** (иначе он не сможет тебе
   писать).

---

## Шаг 2. Вписать токен и Id в настройки

На сервере:
```bash
cd ~/vpn-dimosi-claude
cp -n .env.example .env
nano .env
```
В открывшемся редакторе найди две строки и впиши свои значения (без кавычек):
```
TG_BOT_TOKEN=123456789:AAH...xyz
TG_CHAT_ID=987654321
```
Сохрани и выйди: **Ctrl+X**, затем **Y**, затем **Enter**.

**Проверь, что уведомления приходят:**
```bash
source .env && curl -s "https://api.telegram.org/bot${TG_BOT_TOKEN}/sendMessage" \
  -d chat_id="${TG_CHAT_ID}" -d text="✅ Проверка: уведомления от VPN работают"
```
Если в Telegram пришло сообщение — всё настроено верно.

---

## Шаг 3. Включить автопроверку (cron)

Эта команда заставит сервер каждые 5 минут проверять себя и доступность IP из РФ,
а при проблеме — присылать алерт:
```bash
cd ~/vpn-dimosi-claude
( crontab -l 2>/dev/null;
  echo "*/5 * * * * cd $HOME/vpn-dimosi-claude && ./scripts/healthcheck.sh >/var/log/vpn-health.log 2>&1";
  echo "*/10 * * * * cd $HOME/vpn-dimosi-claude && ./scripts/check-ip-russia.sh >/var/log/vpn-ipcheck.log 2>&1"
) | crontab -
crontab -l        # покажет, что задания добавлены
```

Готово. Теперь:
- **healthcheck** (каждые 5 мин) — следит, что Xray и Hysteria2 запущены и порты
  слушаются; при падении сервиса пришлёт алерт.
- **check-ip-russia** (каждые 10 мин) — проверяет доступность твоего IP с российских
  узлов; если IP перестал отвечать (вероятная блокировка) — пришлёт предупреждение.

> Уведомления приходят **только при проблеме** — спама не будет.

---

## Что делать, когда пришёл алерт

1. «IP недоступен из РФ» → вероятно, заблокирован IP. План — в
   [07-when-blocked.md](07-when-blocked.md): переключись в Shadowrocket на резерв
   `dimosi-hy2`, а если не помогает — подними новый сервер.
2. «Сервис остановлен» → зайди на сервер и выполни `sudo ./deploy.sh health` —
   увидишь, что именно упало; обычно помогает `systemctl restart xray` (или
   `hysteria-server`).

## Отключить / изменить мониторинг

```bash
crontab -e        # откроется список заданий; удали ненужные строки, сохрани
```

## Дополнительно: бэкап раз в неделю в Telegram

Если в `.env` заданы `TG_BOT_TOKEN`/`TG_CHAT_ID`, можно автоматически слать
бэкап настроек себе в Telegram раз в неделю:
```bash
( crontab -l 2>/dev/null;
  echo "0 4 * * 1 cd $HOME/vpn-dimosi-claude && ./scripts/backup.sh >/var/log/vpn-backup.log 2>&1"
) | crontab -
```
