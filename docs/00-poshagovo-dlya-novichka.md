# 00. Полная пошаговая инструкция «повторить всё с нуля»

> Это **проверенный** порядок действий — ровно то, что было проделано при настройке,
> с уже учтёнными подводными камнями. Написано для не-программиста: каждый шаг —
> «что сделать» и «что должно получиться». Следуя ему, ты поднимешь рабочий VPN на
> **любом новом сервере** примерно за 15 минут.

**Что в итоге получится:**
- основной канал **VLESS + Reality + Vision** (маскируется под `www.microsoft.com`);
- резервный быстрый канал **Hysteria2** (UDP + port hopping);
- бэкап для восстановления за 15 минут при блокировке IP.

---

## Шаг 1. Купить сервер (VPS)

**VPS** — арендованный компьютер за границей, работает 24/7.

| Параметр | Что брать |
|----------|-----------|
| Страна | Финляндия, Германия, Нидерланды, Швеция (близко к РФ = быстро) |
| Тариф | 1 CPU / 1 ГБ RAM / 10+ ГБ диска (~3–6 €/мес) |
| Система (OS) | **Ubuntu 24.04** (или 22.04 / Debian 12), **чистый** образ |

> Не бери «панельные»/«VPN-готовые» образы и самые дешёвые «перепроданные» серверы —
> у них часто уже заблокированный («грязный») IP.

**Оплата** часто требует зарубежную карту. После оплаты хостинг пришлёт **IP**,
логин (`root`) и **пароль**.

## Шаг 2. Проверить, «живой» ли IP из России

Открой в браузере (можно с телефона), подставив свой IP:
```
https://check-host.net/check-tcp?host=ТВОЙ_IP:443
```
Смотри узлы с пометкой **RU** (Москва/СПб). Хоть часть зелёные → IP годный.
Все красные → проси у хостинга другой IP или бери другой сервер.

## Шаг 3. Войти на сервер по SSH

**Открой терминал:**
- **Windows:** Пуск → набери `powershell` → открой «Windows PowerShell».
- **Mac:** Cmd+Space → `Терминал` → Enter.
- **Телефон:** приложение **Termius**.

**Подключись** (подставь свой IP) и нажми Enter:
```bash
ssh root@ТВОЙ_IP
```
- Первый раз спросит `Are you sure… (yes/no)` → напечатай `yes`, Enter.
- Спросит пароль. **Печатай его «вслепую»** — на экране НИЧЕГО не появится (ни точек,
  ни звёздочек). Это нормально. Набрал → Enter.
- Увидел строку, оканчивающуюся на `~#` → ты внутри. 🎉

## Шаг 4. Сразу сменить пароль (безопасность)

```bash
passwd
```
Введи новый длинный пароль дважды (тоже вслепую), запиши его в менеджер паролей.

## Шаг 5. Скачать установщик и запустить (одна команда)

Скопируй **весь** блок целиком, вставь в окно SSH (в PowerShell вставка = **правый
клик мышью**) и нажми Enter:
```bash
apt update && apt install -y git
git clone -b claude/russia-vpn-infrastructure-H5PGw https://github.com/dimosiev/vpn-dimosi-claude.git
cd vpn-dimosi-claude
bash bootstrap.sh
```

Что произойдёт автоматически (~2–4 минуты):
1. обновится система, включится файрвол (UFW) и fail2ban;
2. установится Xray и настроится **VLESS + Reality** (основной канал);
3. установится **Hysteria2** (резерв) с port hopping;
4. создастся бэкап в `/root/vpn-backups/`;
5. в конце на экране появятся **ссылка `vless://…` и QR-код**.

> 💡 Установщик можно запускать повторно — он **переиспользует уже созданные ключи**,
> поэтому старые ссылки не ломаются.

## Шаг 6. Подключить телефон

```bash
sudo ./scripts/gen-client-link.sh      # снова показать QR и ссылку
```
1. Установи на телефон **v2RayTun** (App Store / Google Play). Альтернативы: Hiddify, v2rayNG.
2. В приложении: `+` → **Scan QR** → наведи камеру на QR-код в окне SSH → **Connect**.

> ⚠️ **Подключайся через QR, а не копированием текста ссылки** из мессенджеров/чатов —
> там адрес `www.microsoft.com` может «превратиться» в кликабельную ссылку и сломать
> строку. QR всегда корректен. Если копируешь ссылку — только напрямую из окна терминала.

## Шаг 7. Подключить компьютер

- **Windows:** приложение **v2rayN**. **Mac:** **v2RayTun** или **Hiddify**.
- Скопируй ссылку `vless://…` из терминала → в приложении «Import from clipboard» → Connect.

## Шаг 8. Проверить, что всё работает

С включённым VPN:
1. Открой **https://ipinfo.io** — должен показать **IP сервера** (страну хостинга),
   а не твой реальный город. ✅
2. Любой «DNS leak test» — без утечек DNS / WebRTC / IPv6. ✅
3. Спидтест — скорость адекватная. ✅
4. Открой `https://ТВОЙ_IP` **без** VPN — увидишь страницу-донор или TLS-ошибку,
   но не «прокси». Это значит, маскировка работает. ✅

Все пункты ОК → **рабочий устойчивый VPN готов.**

## Шаг 9. Сохранить «тревожный чемоданчик»

```bash
ls -lh /root/vpn-backups/          # тут лежит бэкап vpn-backup-….tar.gz
```
Скачай этот файл к себе на компьютер/флешку (через Termius/WinSCP или попроси меня
показать команду). В нём ключи и настройки — с ним поднимешь всё заново за 15 минут.
Храни копию **оффлайн**.

Также сохрани заранее: установщики клиентов (v2RayTun/Hiddify), **Tor Browser**,
GoodbyeDPI/zapret — на случай, если магазины приложений окажутся недоступны.

## Если перестало работать (блокировка IP)

Это штатная ситуация. Признак: «подключено», но интернета нет; сервер не пингуется из РФ.

1. **Проверь, жив ли IP из РФ:**
   ```bash
   cd ~/vpn-dimosi-claude && ./scripts/check-ip-russia.sh
   ```
2. **IP мёртв** → возьми **новый сервер** (другой хостинг/страна) и повтори Шаги 3–6.
   Чтобы старые ссылки продолжили работать — перенеси на новый сервер папку с ключами:
   на старом `tar -czf state.tgz -C / etc/vpn-dimosi`, скачай, на новом распакуй в
   `/etc/vpn-dimosi`, затем `bash bootstrap.sh` (ключи переиспользуются).
3. **IP жив, но не работает** → переключись на резерв **Hysteria2** в клиенте, либо
   смени донора: в `~/vpn-dimosi-claude` отредактируй и перезапусти Reality (спроси меня).
4. **Заблокировано вообще всё** → Tor Browser + мосты (snowflake/webtunnel).

Подробное дерево решений: [07-when-blocked.md](07-when-blocked.md).

## Шпаргалка команд (на каждый день)

Всё запускать из папки `~/vpn-dimosi-claude`:
```bash
sudo ./deploy.sh status     # статус сервисов + мои ссылки

```

## Подводные камни, с которыми реально столкнулись

| Симптом | Причина | Решение |
|---------|---------|---------|
| Пароль «не вводится» при входе | так и задумано — ввод вслепую | печатай не глядя, жми Enter |
| `Hysteria2 не стартовала` | сервису не хватало прав на чтение конфига | **уже исправлено в установщике**; если повторится: `chown -R hysteria:hysteria /etc/hysteria && chmod 750 /etc/hysteria && chmod 640 /etc/hysteria/* && systemctl restart hysteria-server` |
| Ссылка `vless://` не работает при копировании из чата | мессенджер «съел» часть адреса | подключайся через **QR** или копируй из терминала |
| `Connection refused / timed out` при входе | опечатка в IP или сервер ещё «просыпается» | проверь IP, подожди 2–3 минуты |
| Панель/порт «не открывается» | закрыт в файрволе | `ufw status`, при нужде `ufw allow ПОРТ` |

## Что под капотом (коротко, чтобы понимать)

- **VLESS + Reality** не прячет туннель, а **притворяется визитом на microsoft.com**:
  для DPI это обычная HTTPS-сессия, которую нельзя заблокировать, не сломав сам сайт.
- **Ключи** (UUID, Public Key, Short ID, SNI) должны совпадать на сервере и клиенте —
  ими «свой» клиент отличается от случайного сканера (которого сервер проксирует на
  настоящий microsoft.com).
- **Hysteria2** — другой тип (UDP); ломается DPI иначе, чем Reality, поэтому держим оба.

Полная теория и каталог всех схем (CDN, CFXHTTP, AmneziaWG, Tor): см. остальные
файлы в [`docs/`](.) и [README](../README.md).
