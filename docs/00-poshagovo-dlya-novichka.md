# 00. Главный гайд: с нуля до работающего VPN (для новичка)

> Это **главный документ**. Следуй ему сверху вниз — и поднимешь рабочий VPN на
> **любом новом сервере** примерно за 15 минут, **без посторонней помощи**.
> Написано простым языком: каждый шаг — «что сделать» и «что должно получиться».

**Что в итоге будет:**
- основной канал **VLESS + Reality** (маскируется под `www.microsoft.com`);
- резервный быстрый канал **Hysteria2** (на случай, если основной начнёт лагать);
- бэкап, чтобы восстановиться за 15 минут, если IP заблокируют.

---

## ❗ Сначала — самое важное правило (прочитай!)

Ты будешь работать с **двумя разными компьютерами**, и их легко перепутать:

| Где ты | Как выглядит строка ввода | Какие команды тут работают |
|--------|---------------------------|----------------------------|
| 🖥️ **Твой компьютер** (Mac/Windows) | заканчивается на `%` или `>` (`зовут@MacBook %`) | только `ssh` для входа на сервер |
| ☁️ **Сервер** (после входа по SSH) | заканчивается на `#` (`root@имя:~#`) | все команды из этого гайда: `systemctl`, `./deploy.sh`, `/etc/...` |

> **Если команда отвечает `command not found: systemctl` или `No such file or directory`
> для `/etc/...` — ты на своём компьютере, а не на сервере.** Нужно сначала зайти на
> сервер командой `ssh root@ТВОЙ_IP` (см. Шаг 3) и дождаться приглашения с `#`.

Запомни это — 90% путаницы у новичков именно здесь.

## Шаг 1. Купить сервер (VPS)

**VPS** — арендованный компьютер за границей, работает круглосуточно.

| Параметр | Что брать |
|----------|-----------|
| Страна | Финляндия, Германия, Нидерланды, Швеция (близко к РФ = быстро) |
| Тариф | 1 CPU / 1 ГБ RAM / 10+ ГБ диска (~3–6 €/мес) |
| Система (OS) | **Ubuntu 24.04** (или 22.04 / Debian 12), **чистый** образ |

> Не бери «панельные»/«VPN-готовые» образы и самые дешёвые «перепроданные» серверы —
> у них часто уже заблокированный («грязный») IP.

Оплата часто требует зарубежную карту. После оплаты хостинг пришлёт **IP-адрес**,
логин (`root`) и **пароль**.

## Шаг 2. Проверить, «живой» ли IP из России

Открой в браузере (можно с телефона), подставив свой IP:
```
https://check-host.net/check-tcp?host=ТВОЙ_IP:443
```
Смотри узлы с пометкой **RU** (Москва/СПб). Хоть часть зелёные → IP годный.
Все красные → попроси у хостинга другой IP или возьми другой сервер.

## Шаг 3. Зайти на сервер по SSH

**Открой терминал на своём компьютере:**
- **Windows:** Пуск → набери `powershell` → открой «Windows PowerShell».
- **Mac:** Cmd+Space → набери `Терминал` → Enter.

**Подключись** (подставь свой IP) и нажми Enter:
```bash
ssh root@ТВОЙ_IP
```
- Первый раз спросит `Are you sure… (yes/no)` → напечатай `yes`, Enter.
- Спросит пароль. **Печатай его «вслепую»** — на экране НИЧЕГО не появится (ни точек,
  ни звёздочек). Это нормально, так задумано. Набрал → Enter.
- Увидел строку, оканчивающуюся на `#` (`root@имя:~#`) → **ты на сервере.** 🎉

## Шаг 4. Сразу сменить пароль (30 секунд)

На сервере:
```bash
passwd
```
Введи новый длинный пароль дважды (тоже вслепую), запиши его в менеджер паролей.

## Шаг 5. Скачать код и запустить установку (одна команда)

> Репозиторий должен быть **публичным** на момент скачивания (как сделать —
> см. раздел «Приватность репозитория» ниже). Если он у тебя приватный — на 2 минуты
> переключи на Public, скачай, верни обратно.

Скопируй **весь** блок целиком, вставь в окно SSH (в PowerShell вставка = **правый
клик мышью**; в Терминале Mac = Cmd+V) и нажми Enter:
```bash
apt update && apt install -y git
git clone -b claude/russia-vpn-infrastructure-H5PGw https://github.com/dimosiev/vpn-dimosi-claude.git
cd vpn-dimosi-claude
bash bootstrap.sh
```

Что произойдёт автоматически (~2–4 минуты):
1. обновится система, включится файрвол (UFW) и fail2ban;
2. установится Xray и настроится **VLESS + Reality** (основной канал);
3. установится **Hysteria2** (резерв) с port hopping;
4. создастся бэкап в `/root/vpn-backups/`;
5. в конце на экране появятся **ссылка `vless://…` и QR-код**.

> 💡 Установщик можно запускать повторно — он **переиспользует уже созданные ключи**,
> поэтому старые ссылки не ломаются.

## Шаг 6. Подключить телефон (Shadowrocket)

Подробная инструкция по Shadowrocket с картинками шагов — в
[06-clients.md](06-clients.md). Кратко:

1. На сервере покажи QR основного канала:
   ```bash
   sudo ./scripts/gen-client-link.sh
   ```
2. В **Shadowrocket**: нажми **«+»** вверху справа → значок **QR** → наведи на QR в
   окне терминала. Профиль `dimosi-reality` добавится.
3. Переключи тумблер вверху Shadowrocket в **ON**.
4. (Резерв) Точно так же добавь второй профиль Hysteria2 — QR ссылки
   `hysteria2://…` (её печатает `bootstrap.sh` в самом конце). Получишь два профиля:
   **основной** `dimosi-reality` и **резерв** `dimosi-hy2`, переключаются одним тапом.

> ⚠️ **Подключайся через QR, а не копированием текста ссылки** из мессенджеров —
> там адрес `www.microsoft.com` может «превратиться» в кликабельную ссылку и сломать
> строку. QR всегда корректен. Если копируешь ссылку — только прямо из окна терминала.

## Шаг 7. Проверить, что всё работает

С включённым VPN на телефоне открой в Safari:
1. **https://ipinfo.io** — должен показать **IP сервера** (страну хостинга), а не твой
   город. ✅
2. Любой «DNS leak test» — без утечек. ✅
3. Спидтест — скорость адекватная. ✅

Сменился IP на серверный → **рабочий VPN готов.**

## Шаг 8. Сохранить «тревожный чемоданчик»

```bash
ls -lh /root/vpn-backups/          # тут лежит бэкап vpn-backup-….tar.gz
```
Скачай этот файл к себе на компьютер/флешку (как — см. [05-backup-restore.md](05-backup-restore.md))
и храни **оффлайн**. С ним поднимешь всё заново за 15 минут. Заранее сохрани также
установщики клиентов и **Tor Browser** — на случай, если магазины станут недоступны.

## Шаг 9 (по желанию). Включить мониторинг с алертами

Чтобы сервер сам проверял себя и **писал тебе в Telegram**, если IP заблокируют —
см. [10-monitoring.md](10-monitoring.md). Это лучший способ «узнать о блокировке
первым», а не от того, что «интернет вдруг пропал».

## Приватность репозитория (Public ↔ Private)

В репозитории **нет секретов** (ключи и пароли создаются на сервере; `.env` и `out/`
исключены из git), поэтому он может быть как публичным, так и приватным — на
безопасность твоего VPN это не влияет.

**Сделать приватным** (для спокойствия):
1. Открой `https://github.com/dimosiev/vpn-dimosi-claude/settings`.
2. Внизу, в красной зоне **«Danger Zone»** → **«Change visibility»** → **Private**.

**Когда понадобится новый сервер** (а репозиторий приватный) — выбери любой способ:
- **Проще всего:** на 2 минуты верни видимость на **Public**, выполни `git clone`
  на новом сервере (Шаг 5), затем снова поставь **Private**.
- **Или** восстановись из оффлайн-бэкапа (в нём твои ключи) + скачай код архивом
  (на GitHub: кнопка `Code` → `Download ZIP`, будучи залогиненным).

## Если перестало работать (блокировка IP)

Штатная ситуация. Признак: «подключено», но интернета нет; сервер не пингуется из РФ.

1. **Проверь, жив ли IP из РФ:**
   ```bash
   cd ~/vpn-dimosi-claude && ./scripts/check-ip-russia.sh
   ```
2. **IP мёртв** → бери **новый сервер** и повтори Шаги 3–6. Чтобы старые ссылки
   работали — перенеси папку с ключами (детали в [05-backup-restore.md](05-backup-restore.md)).
3. **IP жив, но не работает** → переключись в Shadowrocket на резерв `dimosi-hy2`.
4. **Заблокировано вообще всё** → Tor Browser + мосты.

Подробное дерево решений: [07-when-blocked.md](07-when-blocked.md).

## Шпаргалка команд (на каждый день)

Сначала зайди на сервер (`ssh root@ТВОЙ_IP`), потом всё запускай из `~/vpn-dimosi-claude`:
```bash
sudo ./deploy.sh status     # статус каналов + мои ссылки
sudo ./deploy.sh health     # проверка: всё ли работает на сервере
./deploy.sh check           # доступен ли сервер из РФ
sudo ./deploy.sh backup     # сделать свежий бэкап
sudo ./scripts/gen-client-link.sh   # снова показать QR и ссылку
```

## Подводные камни (реальные, с которыми сталкиваются)

| Симптом | Причина | Решение |
|---------|---------|---------|
| `command not found: systemctl` / `No such file…/etc/...` | ты на **своём компьютере**, а не на сервере | зайди на сервер: `ssh root@ТВОЙ_IP`, дождись `#`, повтори команду |
| Пароль «не вводится» при входе | так задумано — ввод вслепую | печатай не глядя, жми Enter |
| `Hysteria2 не стартовала` | сервису не хватило прав на конфиг | `chown -R hysteria /etc/hysteria && chmod 750 /etc/hysteria && chmod 640 /etc/hysteria/* && systemctl restart hysteria-server` (уже исправлено в установщике) |
| Ссылка `vless://` не работает при копировании из чата | мессенджер «съел» часть адреса | подключайся через **QR** или копируй из терминала |
| `Connection refused / timed out` при входе | опечатка в IP или сервер «просыпается» | проверь IP, подожди 2–3 минуты |

## Что под капотом (коротко, чтобы понимать, а не бояться)

- **VLESS + Reality** не прячет туннель, а **притворяется визитом на microsoft.com**:
  для систем фильтрации это обычная HTTPS-сессия, которую нельзя заблокировать, не
  сломав сам сайт. Случайный сканёр без твоих ключей просто перенаправляется на
  настоящий microsoft.com и уходит ни с чем.
- **Hysteria2** — другой тип канала (UDP). Фильтры ломают его иначе, чем Reality,
  поэтому два разнотипных канала = меньше шансов остаться совсем без связи.

работы достаточно этого документа.
