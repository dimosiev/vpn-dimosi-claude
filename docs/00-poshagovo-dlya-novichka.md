# 00. Пошаговая инструкция для не-программиста (гибрид-режим)

Эта инструкция написана так, чтобы её мог пройти человек, который **никогда не
настраивал сервер**. Каждый шаг — это «что нажать» и «что должно получиться».
Если что-то пошло не так — копируешь то, что вывел экран, и присылаешь мне.

> **Главная мысль:** 95% работы делает одна команда `sudo ./deploy.sh all`.
> Всё остальное — это «купить сервер» и «войти в него». Не бойся: сломать ничего
> нельзя, сервер дешёвый и одноразовый — в худшем случае удаляешь и берёшь новый.

---

## Шаг 0. Купить сервер (VPS)

**VPS** — это арендованный компьютер за границей, который работает 24/7. Мы
поставим на него наш VPN.

### Что выбрать при заказе

| Параметр | Что брать | Почему |
|----------|-----------|--------|
| Страна (локация) | Финляндия, Германия, Нидерланды или Швеция | близко к РФ = быстро |
| Тариф | 1 CPU, 1 ГБ RAM, 10+ ГБ диска | хватает с большим запасом |
| Операционная система | **Ubuntu 24.04** (или 22.04 / Debian 12) | для неё всё отлажено |
| Образ | **чистый**, НЕ «панельный»/«VPN-готовый» | всё ставим сами |

**Бюджет:** ~3–6 € в месяц. Слишком дешёвые серверы часто имеют «грязный» IP,
который уже заблокирован.

### Про оплату

Часто нужна **зарубежная карта** — российские карты у заграничных хостингов
нередко не проходят. Если карты нет — некоторые хостинги принимают криптовалюту
или другие способы; это уточняется на сайте хостинга.

### После оплаты

Хостинг пришлёт (обычно на почту или в личный кабинет):
- **IP-адрес** сервера — четыре числа через точку, например `192.0.2.10`;
- **логин** — почти всегда `root`;
- **пароль** (или попросит загрузить SSH-ключ — если так, скажи мне, помогу).

### Сразу проверь, «живой» ли IP из России

Это важно: если IP уже заблокирован в РФ, дальше настраивать бессмысленно.
Открой в браузере (можно с телефона):

```
https://check-host.net/check-tcp?host=ТВОЙ_IP:443
```

Нажми кнопку проверки, посмотри на узлы с пометкой **RU** (Москва/СПб). Если хотя
бы часть из них зелёные/отвечают — IP живой, едем дальше. Если все красные —
напиши хостингу «дайте другой IP» или возьми другой сервер.

> 💬 **Мне ничего секретного присылать не нужно.** В гибрид-режиме пароль остаётся
> только у тебя. Мне достаточно знать IP (он не секрет) и видеть, что выводит экран.

## Шаг 1. Войти в сервер по SSH

**SSH** — это способ управлять сервером текстовыми командами.

### Windows

1. Нажми «Пуск» → набери `PowerShell` → открой его.
2. Напиши команду (вместо `ТВОЙ_IP` — адрес сервера) и нажми Enter:
   ```
   ssh root@ТВОЙ_IP
   ```
3. Первый раз спросит `Are you sure… (yes/no)` — напечатай `yes`, Enter.
4. Спросит пароль. **Печатай его «вслепую»** — на экране не появятся даже звёздочки,
   это нормально. Напечатал → Enter.

### Mac / Linux

То же самое, но открой программу **Терминал** и введи `ssh root@ТВОЙ_IP`.

### Телефон

Установи приложение **Termius** (есть для iOS/Android), создай подключение:
Host = IP, Username = root, Password = пароль.

### Как понять, что ты внутри

Появится строка вида `root@hostname:~#`. Поздравляю — ты на сервере. Всё, что
печатаешь дальше, выполняется на нём.

> 🆘 Если пишет `Connection refused` / `timed out` — чаще всего опечатка в IP или
> сервер ещё не «проснулся» после покупки (подожди 2–3 минуты). Пришли мне текст
> ошибки — разберёмся.

## Шаг 2. Скачать наши скрипты на сервер

Скопируй и вставь в окно SSH **целиком** (правый клик в PowerShell = вставка):

```bash
apt update && apt install -y git
git clone <ССЫЛКА_НА_РЕПОЗИТОРИЙ> vpn-dimosi
cd vpn-dimosi
```

> Я дам тебе точную `<ССЫЛКА_НА_РЕПОЗИТОРИЙ>` в чате.

Должно скачаться без ошибок и появиться приглашение, оканчивающееся на `vpn-dimosi#`.

---

## Шаг 3. Настроить параметры (1 минута)

```bash
cp .env.example .env
nano .env
```

Откроется простой текстовый редактор. Для базовой настройки **менять почти ничего
не нужно** — значения по умолчанию рабочие (донор `www.microsoft.com`, порт 443).
Если хочешь уведомления в Telegram — впишешь токен бота позже, это необязательно.

Выйти из nano: `Ctrl + X`, затем `Y` (да, сохранить), затем `Enter`.

---

## Шаг 4. Запустить установку — одна команда

```bash
sudo ./deploy.sh all
```

Эта команда сама:
1. защитит сервер (файрвол, fail2ban);
2. поставит Xray и настроит **VLESS + Reality** (основной канал);
3. поставит **Hysteria2** (быстрый резерв);
4. сделает первый бэкап;
5. в конце **покажет ссылку `vless://…` и QR-код** прямо в окне.

Установка занимает пару минут. **Скопируй весь вывод и пришли мне** — я проверю,
что всё поднялось правильно, и подскажу, если что-то требует внимания.

---

## Шаг 5. Подключить телефон и компьютер

1. Установи приложение-клиент:
   - **iPhone:** v2RayTun (или Hiddify) из App Store;
   - **Android:** v2RayTun или Hiddify из Google Play;
   - **Windows:** v2rayN; **Mac:** v2RayTun.
2. На телефоне: в приложении `+` → **Scan QR** → наведи камеру на QR-код в окне SSH.
3. На компьютере: скопируй ссылку `vless://…` → в приложении «Import from clipboard».
4. Нажми **Connect / Подключить**.

---

## Шаг 6. Проверить, что всё работает

С включённым VPN открой по очереди:
1. `https://ipinfo.io` — должен показать **IP твоего сервера** (страну хостинга),
   а не твой реальный город. ✅
2. Любой «DNS leak test» — не должно быть утечек. ✅
3. Спидтест — скорость адекватная. ✅

Если IP сменился на серверный — **поздравляю, VPN работает.**

---

## Шаг 7. Обезопасить сервер (важно, 30 секунд)

Раз уж мы заходили под root с паролем — смени его на новый длинный. На сервере:

```bash
passwd root
```

Введи новый пароль дважды (опять «вслепую»). Запиши его в надёжное место
(менеджер паролей). Готово.

---

## Шаг 8. Сохранить «тревожный чемоданчик»

Бэкап уже создан на шаге 4. Скачай его себе на компьютер/флешку, чтобы при
блокировке IP поднять всё заново за 15 минут:

```bash
ls -lh /root/vpn-backups/
```

Файл `vpn-backup-….tar.gz` скачай к себе (через приложение типа WinSCP/Termius
или попроси меня показать команду под твою систему). Это и есть твоя страховка —
в нём ключи и настройки, а старые ссылки продолжат работать на новом сервере.

## Что делать, когда «перестало работать»

Это **штатная ситуация**, не поломка. Полный план — в
[07-when-blocked.md](07-when-blocked.md). Кратко:

1. Проверь, жив ли IP из РФ: `./scripts/check-ip-russia.sh`.
2. Если IP мёртв → заведи новый сервер, восстановись из бэкапа (Шаг 8 в обратную
   сторону), запусти `sudo ./deploy.sh all` — ключи переиспользуются.
3. Если IP жив, но не работает → переключись на резервную схему (Hysteria2 уже
   стоит) или смени донора. Пиши мне — проведу.
