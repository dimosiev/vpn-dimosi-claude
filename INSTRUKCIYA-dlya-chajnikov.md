# 🚀 Как поднять VPN с нуля — инструкция для чайников

> Эта инструкция написана так, будто ты делаешь всё **в одиночку и впервые**.
> Читай как чек-лист и выполняй по порядку. **Сломать ничего нельзя** — в худшем
> случае удаляешь сервер и берёшь новый.
>
> Нужна более глубокая версия с пояснениями «почему так» — см.
> [docs/00-poshagovo-dlya-novichka.md](docs/00-poshagovo-dlya-novichka.md).

**Тебе понадобится:** банковская карта (желательно зарубежная), компьютер, телефон, ~20 минут.

## 🧭 Золотое правило (запомни сразу)

Строка ввода в окне терминала подсказывает, **где ты находишься**:

| Конец строки | Где ты | Что тут работает |
|---|---|---|
| `#` (`root@имя:~#`) | **на сервере** | все команды установки |
| `%` или `>` | **на своём компьютере** | только `ssh` для входа на сервер |

Если команда отвечает `command not found: systemctl` или ошибку про `/etc/...` —
**ты на своём компьютере**, надо сначала зайти на сервер (Часть 3).

# ЧАСТЬ 1. Купить сервер (5 минут)

1. Зайди на сайт любого хостинга с серверами в Европе (Финляндия / Германия / Нидерланды).
2. Выбери самый дешёвый тариф **VPS**: примерно 1 CPU, 1–2 ГБ RAM, 10+ ГБ диска (~3–6 €/мес).
3. В поле «Операционная система / Image» выбери **Ubuntu 24.04** (или 22.04).
   **Не** бери образы со словами «VPN», «panel», «3x-ui» — нужен чистый.
4. Оплати. Через 1–5 минут хостинг пришлёт (на почту или в личный кабинет):
   - **IP-адрес** — 4 числа через точку, например `62.60.246.254`;
   - **логин** — `root`;
   - **пароль** — набор букв и цифр.
5. **Запиши эти три вещи** куда-нибудь рядом.

---

# ЧАСТЬ 2. Проверить, что IP «живой» из России (1 минута)

1. Открой в браузере, подставив свой IP вместо `ТВОЙ_IP`:
   ```
   https://check-host.net/check-tcp?host=ТВОЙ_IP:443
   ```
2. Нажми кнопку проверки, подожди 10 секунд.
3. Смотри на строки с пометками **RU** (Москва, СПб):
   - есть зелёные / со временем отклика → **IP годный, продолжаем**;
   - все красные → попроси у хостинга другой IP или возьми другой сервер.

# ЧАСТЬ 3. Зайти на сервер (3 минуты)

### Windows
1. Нажми «Пуск» (значок Windows внизу слева).
2. Набери на клавиатуре: `powershell`.
3. Кликни «Windows PowerShell» — откроется синее окно.

### Mac
1. Нажми `Cmd + Пробел` (поиск Spotlight).
2. Набери `Терминал`, нажми Enter — откроется окно.

### Дальше одинаково (в этом окне):
4. Напечатай команду (подставь свой IP) и нажми Enter:
   ```
   ssh root@ТВОЙ_IP
   ```
5. **Первый раз** спросит `Are you sure you want to continue connecting (yes/no)?` —
   напечатай `yes`, нажми Enter.
6. Появится запрос пароля. Напечатай пароль от сервера.
   > ⚠️ При вводе пароля на экране **НИЧЕГО не появляется** — ни точек, ни звёздочек.
   > **Это нормально!** Печатай вслепую и нажми Enter. Ошибся — увидишь
   > `Permission denied`, просто введи команду из п.4 заново.
7. **Ты внутри**, когда видишь строку, оканчивающуюся на `#`, например
   `root@technological-ha:~#`. 🎉

---

# ЧАСТЬ 4. Сменить пароль (30 секунд)

На сервере (там, где `#`):
1. Напечатай и нажми Enter:
   ```
   passwd
   ```
2. Введи **новый** пароль (вслепую) → Enter.
3. Введи его же ещё раз → Enter.
4. Увидишь `password updated successfully`. **Запиши новый пароль** в надёжное место.

---

# ЧАСТЬ 5. Скачать и запустить установку (5 минут)

### Если репозиторий в Private — открой его на пару минут
1. Открой в браузере: `https://github.com/dimosiev/vpn-dimosi-claude/settings`
2. Прокрути в самый низ → **«Danger Zone»** → **«Change visibility»** → **Public** → подтверди.
   *(Если репозиторий и так публичный — пропусти этот пункт.)*

### Установка
1. Вернись в окно терминала (где `#`).
2. **Скопируй этот блок целиком** (4 строки) и вставь в окно:
   - в **PowerShell**: **правый клик мышью** = вставка;
   - на **Mac**: `Cmd + V`.
   ```bash
   apt update && apt install -y git
   git clone https://github.com/dimosiev/vpn-dimosi-claude.git
   cd vpn-dimosi-claude
   bash bootstrap.sh
   ```
3. Нажми **Enter** и **жди 2–4 минуты** (по экрану побегут строки — это нормально).
4. В конце увидишь `[✓] ГОТОВО. Инфраструктура развёрнута.`, а ниже — **QR-код** и
   ссылки `vless://…` / `hysteria2://…`. **Оставь окно открытым.**

> 🔒 После установки можешь вернуть репозиторий обратно в **Private** — код уже на сервере.

# ЧАСТЬ 6. Подключить телефон (Shadowrocket) (2 минуты)

1. Установи на телефон приложение **Shadowrocket** (App Store).
2. Если QR «уехал» вверх — на сервере покажи его снова:
   ```
   sudo ./scripts/gen-client-link.sh
   ```
3. Открой **Shadowrocket** → нажми **«+»** в правом верхнем углу.
4. Нажми **значок QR-кода** в строке адреса.
5. Наведи камеру на QR-код в окне терминала → профиль `dimosi-reality` добавится.
6. На главном экране переключи **большой тумблер вверху** в **ON** (станет зелёным).

*(По желанию повтори п.3–5 для ссылки `hysteria2://…` — получишь резервный профиль `dimosi-hy2`.)*

---

# ЧАСТЬ 7. Проверить, что работает (1 минута)

С включённым Shadowrocket, на телефоне в Safari:
1. Открой `https://ipinfo.io`
2. Должно показать **IP твоего сервера** и страну хостинга, **а не твой город**.
   - Серверный IP → 🎉 **VPN работает!**
   - Твой город → VPN не включился: проверь тумблер ON и выбранный профиль.

---

# ЧАСТЬ 8. Сохранить «страховку» (1 минута)

1. На сервере:
   ```
   ls -lh /root/vpn-backups/
   ```
   Увидишь файл `vpn-backup-….tar.gz` — копия с твоими ключами.
2. Сохрани его себе (через Termius / WinSCP или просто помни, где он лежит).
   С ним при блокировке IP поднимешь всё на новом сервере за 15 минут.

# 🆘 Если что-то пошло не так

| Что видишь | Что значит | Что делать |
|---|---|---|
| `command not found: systemctl` / ошибка про `/etc/...` | ты на **своём компьютере**, не на сервере | `ssh root@ТВОЙ_IP`, дождись `#`, повтори команду |
| Пароль «не печатается» | так и должно быть (ввод скрыт) | печатай вслепую, жми Enter |
| `Permission denied` при входе | опечатка в пароле | введи `ssh root@ТВОЙ_IP` заново |
| `Connection timed out` | опечатка в IP или сервер «просыпается» | проверь IP, подожди 2–3 мин |
| `Could not resolve host` при `git clone` | репозиторий приватный или опечатка | открой репозиторий в Public (Часть 5) |
| `Hysteria2 не стартовала` в конце | мелочь, основной VPN работает | `chown -R hysteria /etc/hysteria && chmod 750 /etc/hysteria && chmod 640 /etc/hysteria/* && systemctl restart hysteria-server` |
| На телефоне IP не сменился | VPN не подключился | проверь тумблер ON и выбранный профиль |

# 🔄 ЧАСТЬ 9. Держать свежим (раз в пару недель)

DPI постоянно «умнеет», но и Xray/Hysteria постоянно улучшают обход. Свежие версии
часто проходят там, где старые уже режут. Обновить всё — **одна команда** (на сервере):
```bash
cd ~/vpn-dimosi-claude
sudo ./deploy.sh update
```
Она сама: сделает бэкап → подтянет последние Xray и Hysteria2 → перезапустит →
покажет версии «было → стало». Ключи и ссылки **не меняются**.

Узнать, есть ли что-то новое (ничего не трогая):
```bash
sudo ./deploy.sh versions
```
Подробнее: [docs/11-obnovleniya.md](docs/11-obnovleniya.md).

# 👥 ЧАСТЬ 10. Поделиться с коллегами

В репозитории **нет твоих секретов** — им можно спокойно делиться. У каждого
коллеги будет **свой отдельный VPN со своими ключами**.

1. Скинь коллеге ссылку на этот репозиторий (он должен быть **Public**).
2. Коллега берёт **свой** сервер и идёт по этой же инструкции с Части 1.
3. Через ~15 минут у него свой рабочий VPN.

---

**Весь путь — 8 частей, ~20 минут.** Готово — ты сам поднял VPN. 💪
